# 机核 - 塔防卡牌对战游戏开发计划

## 项目概述

- **项目名称**: 机核
- **项目类型**: 单机卡牌对战 + Roguelike + 塔爬层机制
- **目标平台**: PC (后续可扩展至移动端)
- **版本**: V1.9
- **核心玩法**: 以"斗兽棋"为基础的卡牌对战 + 7层塔爬层挑战

---

## 一、核心游戏循环

```
开始游戏 → 新手教程 → 选择Buff(3选1) → 进入第1层
    ↓
每日循环:
  - 每日可选: 访问原住民，每日1次
  - 每日可选: 访问主角房间，3选1访问或战斗，每日1次
  - 随时可做: 商店购物、查看已解锁对话
  - 每日可选：挑战BOSS
  - 确认结束: 进入下一天
    ↓
挑战BOSS → 对战胜利 → 获得奖励 → 进入下一层
    ↓
直到: 通关7层(胜利) 或 超过3天/层(失败)
```

## 二、游戏机制详细规范

### 2.1 塔系统

| 属性     | 值                  |
| ------ | ------------------ |
| 总层数    | 7层                 |
| 每层停留上限 | 3天                 |
| 初始资金   | 50金币               |
| 失败条件   | 第3天结束或挑战BOSS失败 |


**层数进度**: 玩家当前层数记录，每层有独立BOSS和NPC

### 2.2 卡牌对战系统

#### 2.2.1 核心规则

- **对战基础**: 基于"斗兽棋"规则
- **卡牌上限**: 每局最多携带10张牌
- **必含卡牌**: 必须包含1张国王 + 1张平民
- **唯一性**: 每位玩家只能拥有1张国王和1张平民

#### 2.2.2 等级克制链

```
等级: 1 < 1.1 < 1.5 < 2 < 2.5 < 3 < 1 < 3.5

特殊规则:
- 3.5级卡可战胜1级卡
- 1级卡(平民)可战胜3级卡(国王)
- 其他情况下1级卡不能战胜国王
```

#### 2.2.3 对战流程

1. 可选择: 查看卡组、查看弃牌堆、出牌、使用一次性卡牌
2. 双方同时出牌
3. 比较等级，败者卡牌移入弃牌堆
4. 平局: 双方卡牌都移入弃牌堆
5. 重复直到一方手牌耗尽或达到回合上限

#### 2.2.4 透视能力（视界限）

- **初始值**: 可看穿敌人1张牌
- **升级**: 每成功爬到下一层，增加1张透视
- **效果**: 战斗开始前随机选定敌人的一张牌，其在战斗过程中全程对玩家保持可见的透视状态，无论其在手牌或者出牌区

#### 2.2.5 回合上限

- **计算公式**: 上限回合数 = (己方初始卡牌数 + 敌方初始卡牌数) / 2
- 达到上限时，总手牌等级低的一方获胜
- 特殊情况：**天气"暖风"**: 移除本局回合上限

#### 2.2.6 胜利奖励

- 战胜后可选获得敌人1张卡牌(国王/平民除外)
- 通关驻守者时获得额外奖励
    - 奖励选项: 
        - 额外金币=5金币X**剩余天数**
        - 5选1 (可选择金币或其他卡牌)

#### 2.2.7 限制规则

- 不能连续两次出同一张牌
- 已出且败北的牌不可再使用（进入弃牌堆了）

### 2.3 天气系统


| 天气     | 效果             |
| ------ | -------------- |
| 雨季     | 获得金钱+50%       |
| 烈日     | 所有.5等级卡牌-0.5   |
| 冰雹     | 所有.5等级卡牌+0.5   |
| 暖风     | 移除本局回合上限       |
| 终焉 | 敌人完全预测了你的出牌，将会刚好选择他的卡组中比你大且差值最小的那一张进行出牌，如果没有，随机出。 |


### 2.4 【此部分待最终确认】 Roguelike Buff系统

**初始选择**: 进入塔时从以下Buff中3选1


| #   | Buff名称 | 效果                    |
| --- | ------ | --------------------- |
| 1   | 社交达人   | 每天访问人物多一次对话机会         |
| 2   | 富二代     | 初始资金+50%              |
| 3   | 无限供应   | 商店物品不会卖光              |
| 4   | 随机平民   | 平民卡每次对弈能力值随机(2~0)     |
| 5   | 随机国王   | 国王卡每次对弈能力值随机(4~0)     |
| 6   | 惊喜卡包   | 获得卡牌数量在0~3中随机         |
| 7   | 盗贼本能   | 拥有"偷盗"技能(可能被暴打)       |
| 8   | 透视增强   | 初始可看穿敌人2张牌            |
| 9   | 混乱战场   | 与非BOSS战斗完全随机，但开局自选3张卡 |

**后续选择**：每爬2层可以再次3选1Buff，已选过的Buff不能再选

总结：
- 第1层（初始），第3层，第5层，第7层开始时可以抽取3个Buff，再从这3个Buff中选择一个生效；
- 做不放回抽取，生效的Buff在本次游戏不会再次出现在Buff池中；

### 2.5 【此部分待最终确认】敌人AI规则

- 敌人为机器人
- 每个敌人牌组<10张
- AI逻辑: 尽可能获胜，不会乱出牌

## 五、剧情与对话系统

### 5.1 剧情系统

#### 5.1.1 角色设定

- 1名玩家角色（主角）
- 3名参赛者（主角团其他成员）
- 8个NPC（每层1个，第1层有2个）

#### 5.1.2 参赛者

- 每名参赛者都有7段剧情对话
- 每天最多与1名参赛者对话
- 对话的剧情由当前层数决定，最多展示到当前层数的对话。比如第3层最多展示3段对话。
- Hero表配置字段：
    - id, 姓名，立绘，头像

### 5.1.3 原住民

- 每个原住民都有3段剧情对话
- 每层获得3次与原住民对话的机会，可以一直保存
- 每天可以与一名已访问的原住民对话（如果能继续推进剧情的话），或者随机访问一名新的原住民
- 随机访问原住民的逻辑：从当前层及之前层的原住民列表里随机抽取一名未访问过的。
- 如果没有未访问过的原住民，就只能与已访问原住民对话
- NPC表配置字段：
    - npcID，姓名，立绘，头像
    - NPCID配置格式：{层数}_{第几个}

### 5.2 对话系统

#### 5.2.1 对话逻辑

- 需要记录每个参赛者的对话进度
- 需要记录每个原住民的对话进度
- 对话配置项：唯一id，背景id，人物id，对话内容，是否选择，下一段id，选项配置
- 【新】通过人物id和人物表找到立绘，人物姓名；
- 唯一id解析逻辑：
    - 参赛者对话：需要解析出对话次数、参赛者槽位。即每一名参赛者的每一段对话对应一个唯一id。
    - 原住民对话：需要解析出对话次数、NPCID。即每一名原住民的每一段对话对应一个唯一id。
- 特殊：旁白配置在Hero表中，人物id是"0"，不需要立绘或姓名。

#### 5.2.2 对话触发时机

- 新手教程对话
- 访问NPC/主角时触发剧情对话
- 战斗前可与敌人交谈(获取出牌线索)
- 已解锁对话可随时回顾

#### 5.2.3 对话跳过

- 直接跳到当前对话最后一句

## 九、技术实现要点

### 8.1 核心数据结构

```csharp
// 玩家数据
class PlayerData {
    int currentFloor;      // 当前层数(1-7)
    int currentDay;       // 当前天数
    int floorDay;         // 本层已待天数
    int gold;             // 金币
    List<Card> handCards; // 手牌
    List<Card> discardPile;// 弃牌堆
    List<Card> ownedCards;// 拥有卡牌
    string selectedBuff;  // 选择的Buff
    int xrayCount;        // 透视次数
    Weather currentWeather;// 当前天气
}

// 卡牌数据
class Card {
    string name;          // 卡牌名称
    float level;          // 等级
    CardType type;        // 基础/功能/能力/一次性
    string effect;        // 效果描述(可留空)
}

// 战斗状态
class BattleState {
    List<Card> playerHand;
    List<Card> enemyHand;
    List<Card> playerDiscard;
    List<Card> enemyDiscard;
    List<Card> playerVisible;  // 敌人可见的我方卡牌
    List<Card> enemyVisible;   // 我方可见的敌人卡牌
    int round;
    int maxRound;             // 回合上限 = (playerHand.Count + enemyHand.Count) / 2
}
```
