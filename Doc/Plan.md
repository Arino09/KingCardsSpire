# 机核 - 塔防卡牌对战游戏开发计划

## 项目概述

- **项目名称**: 机核
- **项目类型**: 单机卡牌对战 + Roguelike + 塔爬层机制
- **目标平台**: PC (后续可扩展至移动端)
- **版本**: V1.7
- **核心玩法**: 以"斗兽棋"为基础的卡牌对战 + 7层塔爬层挑战

---

## 一、核心游戏循环

### 1.1 主循环流程
```
开始游戏 → 选择Buff(3选1) → 进入第1层
    ↓
每日循环:
  - 每日选择: 打工 或 访问原住民房间
  - 每日可选: 访问主角房间
  - 随时可做: 商店购物、查看已解锁对话
  - 确认结束: 进入下一天
    ↓
挑战BOSS → 对战胜利 → 选择获得卡牌 → 进入下一层
    ↓
直到: 通关7层(胜利) 或 资金归0(失败) 或 超过3天/层(失败)
```

### 1.2 每日行动菜单
| 行动 | 效果 | 消耗 |
|------|------|------|
| 挑战BOSS | 进入卡牌对战 | 免费 |
| 前往商店 | 购买卡牌/道具 | 金币 |
| 打工赚钱 | 固定获得金币 | 免费 |
| 访问角色 | 触发剧情/战斗/偷盗事件 | 免费 |

## 二、游戏机制详细规范

### 2.1 塔系统

| 属性 | 值 |
|------|-----|
| 总层数 | 7层 |
| 每层停留上限 | 3天 |
| 初始资金 | 50金币 |
| 失败条件 | 资金归0 或 超过3天未挑战BOSS |

**层数进度**: 玩家当前层数记录，每层有独立BOSS和NPC

### 2.2 卡牌对战系统

#### 2.2.1 核心规则
- **对战基础**: 基于"斗兽棋"规则
- **卡牌上限**: 每局最多携带10张牌
- **必含卡牌**: 必须包含1张国王 + 1张平民
- **唯一性**: 每位玩家只能拥有1张国王和1张平民

#### 2.2.2 等级克制链
```
等级: 1 < 1.1 < 1.5 < 2 < 2.5 < 3 < 1 < 3.5

特殊规则:
- 3.5级卡可战胜1级卡
- 1级卡(平民)可战胜3级卡(国王)
- 其他情况下1级卡不能战胜国王
```

#### 2.2.3 对战流程
1. 双方展示透视可见的卡牌
2. 可选择: 交谈、查看卡组、出牌、使用一次性卡牌
3. 双方同时出牌
4. 比较等级，败者卡牌移入弃牌堆
5. 平局: 双方卡牌都移入弃牌堆
6. 重复直到一方手牌耗尽或达到回合上限

#### 2.2.4 透视能力
- **初始值**: 可看穿敌人1张牌
- **升级**: 每成功爬到下一层，增加1张透视
- **效果**: 战斗开始前可预览敌人部分手牌

#### 2.2.5 回合上限
- 每场对战有最大回合数限制
- 达到上限时，总手牌等级低的一方获胜
- **天气"暖风"**: 移除回合上限

#### 2.2.6 胜利奖励
- 战胜后可选获得敌人1张卡牌(国王/平民除外)

#### 2.2.7 限制规则
- 不能连续两次出同一张牌
- 已出且败北的牌不可再使用

### 2.3 天气系统

| 天气 | 效果 |
|------|------|
| 雨季 | 获得金钱+50% |
| 烈日 | 所有.5等级卡牌-0.5 |
| 冰雹 | 所有.5等级卡牌+0.5 |
| 暖风 | 移除本局回合上限 |
| ~~终焉~~ | ~~塔主特有天气(待定)~~ |

### 2.4 Roguelike Buff系统

**初始选择**: 进入塔时从以下10个Buff中3选1

| # | Buff名称 | 效果 |
|---|----------|------|
| 1 | 社交达人 | 每天访问人物多一次对话机会 |
| 2 | 富二代 | 拥有更多初始资金 |
| 3 | 无限供应 | 商店物品不会卖光 |
| 4 | 随机平民 | 平民卡每次对弈能力值随机(2~0) |
| 5 | 随机国王 | 国王卡每次对弈能力值随机(4~0) |
| 6 | 惊喜卡包 | 获得卡牌数量在0~3中随机 |
| 7 | 高薪工作 | 打工获得更多资金 |
| 8 | 盗贼本能 | 拥有"偷盗"技能(可能被暴打) |
| 9 | 透视增强 | 初始可看穿敌人2张牌 |
| 10 | 混乱战场 | 与非BOSS战斗完全随机，但开局自选3张卡 |

## 八、技术实现要点

### 8.1 推荐技术栈
- **引擎**: Unity (C#) 或 Godot (GDScript/C#)
- **UI**: 可使用FairyGUI或自研UI系统
- **存档**: JSON格式本地存储

### 8.2 核心数据结构

```csharp
// 玩家数据
class PlayerData {
    int currentFloor;      // 当前层数(1-7)
    int currentDay;       // 当前天数
    int floorDay;         // 本层已待天数
    int gold;             // 金币
    List<Card> handCards; // 手牌
    List<Card> discardPile;// 弃牌堆
    List<Card> ownedCards;// 拥有卡牌
    string selectedBuff;  // 选择的Buff
    int xrayCount;        // 透视次数
    Weather currentWeather;// 当前天气
}

// 卡牌数据
class Card {
    string name;          // 卡牌名称
    float level;          // 等级
    CardType type;        // 基础/功能/能力/一次性
    string effect;        // 效果描述(可留空)
}

// 战斗状态
class BattleState {
    List<Card> playerHand;
    List<Card> enemyHand;
    List<Card> playerDiscard;
    List<Card> enemyDiscard;
    List<Card> playerVisible;  // 敌人可见的我方卡牌
    List<Card> enemyVisible;   // 我方可见的敌人卡牌
    int round;
    int maxRound;
}
```

### 8.3 敌人AI实现
1. 评估当前手牌与可见牌
2. 预测玩家可能的出牌
3. 选择最优出牌策略
4. 优先保证胜利，避免送牌

### 8.4 关键算法
- **等级比较**: 根据斗兽棋规则实现比较逻辑
- **天气效果**: 实时修改卡牌等级计算
- **透视系统**: 对战开始时随机暴露X张敌人牌
