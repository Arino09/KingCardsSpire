# 对话配置说明

本文档说明机核项目中 **对话数据表（ScriptableObject）**、**入口 id 与游戏内触发点的对应关系**、**立绘/背景资源约定** 以及 **存档与解锁** 行为，供策划与程序对照维护。

---

## 1. 相关文件与类型

| 类别 | 路径 |
|------|------|
| 对话行/表定义 | `Assets/Scripts/Configs/DialogueConfig.cs`（含 `DialogueLineEntry`、`DialogueChoiceEntry`） |
| 参赛者人物表 | `Assets/Scripts/Configs/HeroConfig.cs`（Addressables 标签 `config_hero`） |
| 原住民人物表 | `Assets/Scripts/Configs/NpcConfig.cs`（Addressables 标签 `config_npc`） |
| 时机类型枚举 | `Assets/Scripts/Configs/DialogueTimingKind.cs` |
| 运行时索引 | `Assets/Scripts/Managers/ConfigManager.cs`（按标签加载 `DialogueConfig` 并合并为 id → 行） |
| Addressables 标签 | `Assets/Scripts/Core/AddressableLabels.cs` 中的 `config_dialogue` |
| 示例配置资产 | `Assets/GameConfigs/Bootstrap/Dialogue_Default.asset` |
| 播放控制器 | `Assets/Scripts/Controllers/DialogueController.cs` |
| 资源地址解析 | `Assets/Scripts/Core/DialogueArtResolver.cs` |
| 对话 UI | `Assets/Scripts/Views/UI/DialogView.cs`，预制体 `Assets/GameAssets/UI/DialogView.prefab` |
| 面板枚举 | `Assets/Scripts/Models/Enums.cs` 中的 `UIPanelId.Dialog`，Addressables 键 `UI/DialogView` |

多个 `DialogueConfig` 资源可同时打上 **`config_dialogue`** 标签；运行时 **所有表中的行按 `Id` 合并到同一字典**，**不同资源里禁止出现重复的 `Id`**（后加载的会覆盖先加载的，以实际 Addressables 加载顺序为准）。

## 2. 表结构：每一行（`DialogueLineEntry`）

| 字段 | 类型 | 说明 |
|------|------|------|
| **Id** | string | 对话行唯一 id，全表不重复。用于链式跳转与入口匹配。 |
| **Timing Kind** | `DialogueTimingKind` 枚举 | **策划分类用**（筛选、文档），**不参与**运行时选入口；见下文 §6。 |
| **Background Id** | string | 背景图逻辑 id，经 `DialogueArtResolver` 转为 Addressables 地址，见 §5。 |
| **Character Id** | string | 人物 id：参赛者查 Hero 表、原住民查 NPC 表；`0` 表示旁白。未命中人物表时才按旧资源 id 兼容解析，见 §5。 |
| **Speaker Name** | string | 编辑器备注/旧配置兼容字段；运行时优先使用 Hero/NPC 表中的姓名。 |
| **Body Text** | string | 正文（可多行）。 |
| **Is Choice** | bool | `true` 为选择支：隐藏「继续」，显示选项列表；`false` 为普通句：点「继续」或「跳过」。 |
| **Next Id** | string | **非选择行**：本句结束后跳转的下一句 **Id**；**留空表示本链结束**（关闭对话面板）。选择行通常不依赖 `Next Id`，由选项的 `Next Dialogue Id` 决定跳转。 |
| **Choices** | 列表 | 仅当 `Is Choice == true` 时使用；每项含 **选项描述**、**跳转对话 Id**。 |

### 2.1 选择支（`DialogueChoiceEntry`）

- **Option Text**：按钮上显示的文字。
- **Next Dialogue Id**：点击后跳转到的下一行 **Id**（必须存在于表中，否则播放会中断并报日志）。

### 2.2 链式剧情写法

1. 为每条链准备 **一条入口行**（见 §3），其 `Id` 必须与代码解析出的 **入口字符串完全一致**。
2. 入口行之后：用 **Next Id** 串起多句非选择对白。
3. 需要分支时：某一行的 `Is Choice = true`，在 `Choices` 里配置多个跳转 `Id`。
4. **结束**：最后一句非选择行的 **Next Id 留空**；或选择支跳转到一句 `Next Id` 为空的行。

## 3. 入口 Id：谁在什么时候播哪一句「第一句」

运行时只认 **`PlayDialogue(startId)`** 传入的 **`startId`**。各系统按下面规则算出 `startId`，再在表里查找 **是否存在 `Id == startId` 的行**；若不存在，行为依具体解析函数而定（见各小节）。

### 3.1 新游戏开场教程

| 项目 | 说明 |
|------|------|
| 触发 | 主菜单 **新游戏**，关闭主菜单后、进入塔内主界面前。 |
| 代码 | `Assets/Scripts/Views/UI/MainMenuView.cs` |
| 固定入口 id | **`tutorial_opening`** |

教程播完后会设置 `PlayerData.HasCompletedOpeningTutorial` 并写档；**继续游戏**不会再次播开场教程（旧档见 `SaveData.Version` 迁移逻辑，`GameManager.RestoreRunFromSave`）。

---

### 3.2 挑战 BOSS 战前

| 项目 | 说明 |
|------|------|
| 触发 | 塔内主界面点击 **挑战 BOSS**，开战与打开战斗界面前。 |
| 代码 | `Assets/Scripts/Views/UI/MainHubView.cs` → `DialogueController.ResolveBossPreFightStartId` |

**解析优先级（从高到低）**，在表中 **命中第一条存在的入口行** 即使用：

1. **`boss_prefight_{BossId}`**  
   - `BossId` 来自当前层的 `FloorState.BossId`（由塔配置 `TowerFloorEntry.BossId` 同步）。  
   - 与塔表里填写的字符串 **完全一致**（区分大小写、下划线等）。

2. **`boss_prefight_f{层}`**  
   - `{层}` 为 **1-based** 当前层数，例如第 7 层 → `boss_prefight_f7`。

3. **`boss_prefight`**  
   - 通用兜底句。

**为每个 BOSS 写独立台词**：在塔配置里保证每层 `BossId` 唯一且稳定，并在对话表中增加 `boss_prefight_你的BossId` 作为入口行即可。

---

### 3.3 访问参赛者（主角房 · 交谈）

| 项目 | 说明 |
|------|------|
| 触发 | 打开主角房，选某一参赛者槽位后点 **交谈**。 |
| 代码 | `Assets/Scripts/Views/UI/HeroView.cs` → `GameManager.TryPrepareHeroDialogue` → `DialogueController.BuildHeroStoryStartId` |
| 槽位 | `0` / `1` / `2`，与界面三个按钮顺序一致。 |

**入口 id**：`hero_{槽位}_{剧情段序号}`。

- 例如槽位 1 的第 2 段剧情 → `hero_1_2`。
- 每名参赛者最多 7 段；当天最多推进 1 名参赛者。
- 可推进段数受当前层数限制：第 3 层最多推进到第 3 段。
- 缺少入口行时不会推进进度，也不会记录当天已交谈。

---

### 3.4 访问原住民（NPC）

| 项目 | 说明 |
|------|------|
| 触发 | 原住民界面中成功准备出下一段剧情入口后，界面内协程播放。 |
| 代码 | `Assets/Scripts/Views/UI/NpcView.cs` → `GameManager.TryPrepareNpcDialogue` / `TryPrepareRandomNewNpcDialogue` |

**入口 id**：`npc_{npcId}_{剧情段序号}`。

- 例如 NPC `1_2` 的第 3 段剧情 → `npc_1_2_3`。
- 每名原住民最多 3 段。
- 每层获得 3 次原住民剧情机会，可跨天保存；每天最多消耗 1 次。
- 随机结识新原住民时，会从当前层及之前层的未访问 NPC 中抽取。
- 缺少入口行时不会推进进度，也不会消耗当天访问或原住民剧情机会。

### 3.5 通关大结局

| 项目 | 说明 |
|------|------|
| 触发 | 第 7 层 BOSS 战胜后，在驻守奖励界面确认奖励并关闭战斗流程，`GameManager.IsRunVictory == true` 时。 |
| 代码 | `Assets/Scripts/Views/UI/CardRewardView.cs` |

## 4. 运行时其它行为

### 4.1 解锁记录

每展示一行（无论是否选择支），会将该行的 **`Id`** 写入 `PlayerData.UnlockedDialogues`（去重），供后续「已解锁对话」类功能使用。实现见 `GameManager.RegisterDialogueLineSeen`。

### 4.2 结构化剧情进度

- `PlayerData.HeroDialogueProgress` 记录每个参赛者已完成的剧情段数。
- `PlayerData.NpcDialogueProgress` 记录每个原住民已完成的剧情段数。
- `PlayerData.LastHeroDialogueDay` / `LastNpcInteractionDay` 记录当天是否已推进对应剧情。
- `PlayerData.NpcDialogueCredits` 记录剩余原住民剧情机会；进入新层时增加 3 次。

### 4.3 跳过

点击 **跳过** 会沿当前非选择链的 `Next Id` 直接跳到最后一句并展示；如果当前后续是选择支或配置链异常，则不会自动替玩家选择分支。

### 4.4 服务注册

`DialogueController` 在 `GameRoot` 启动时通过 `ServiceLocator.Register` 注册；各界面通过 `ServiceLocator.Get<DialogueController>()` 取得实例后，一般将协程挂在 `UIManager` 上 `StartCoroutine(dialogue.PlayDialogue(...))`，避免随临时 UI 销毁而中断。

## 5. 背景与立绘（`DialogueArtResolver`）

地址均按 **Addressables 的 Sprite 主地址** 解析（与楼层背景、NPC 头像等工程约定一致）。

### 5.1 背景（`Background Id`）

- 若 `backgroundId` 不包含子文件夹路径：读取旧楼层背景目录，自动补 **`.jpg`**。例如 `first1` 会解析为 `Sprites/UI/FloorBg/first1.jpg`。
- 若 `backgroundId` 包含子文件夹路径：读取 NPC 背景目录，自动补 **`.jpg`**。例如 `npc_1_1/room` 会解析为 `Sprites/Character/NPCbg/npc_1_1/room.jpg`。
- 若 `backgroundId` 已以 `.jpg` 或 `.png` 结尾：不再自动补扩展名。
- 反斜杠会转为 `/`。

### 5.2 立绘（`Character Id`）

- 若 `Character Id` 为 `0`：按旁白处理，不显示姓名和立绘。
- 若命中 Hero/NPC 表：使用人物表中的 `portraitId` 解析立绘，使用 `displayName` 显示姓名。
- 若未命中人物表：按旧配置兼容，将 `Character Id` 直接作为资源逻辑 id 解析。
- 反斜杠会转为 `/`。
- 若 **已以** `.png` 或 `.jpg` 结尾：  
  `Sprites/Character/{characterId}`
- 否则自动补 **`.png`**：  
  `Sprites/Character/{characterId}.png`

**示例**：`Hero/1` → `Sprites/Character/Hero/1.png`；`NPC/Avatar/0_1.png` → `Sprites/Character/NPC/Avatar/0_1.png`。

对应 Sprite 资源需加入 Addressables，且地址与上述字符串一致（可参考 `Sprites/Character/NPCbg`、`Sprites/Character/...` 分组）。

## 6. 时机类型枚举（`DialogueTimingKind`）

用于在 Inspector 里 **标注本行所属剧情类型**，便于策划筛选与文档统一；**不替代** §3 的入口 id 规则。

| 枚举值 | 含义建议 |
|--------|----------|
| `None` | 未标注 |
| `OpeningTutorial` | 新游戏开场教程 |
| `BossPreFight` | 挑战 BOSS 战前 |
| `HeroVisit` | 访问参赛者 / 主角房交谈 |
| `NpcVisit` | 访问原住民 |
| `StoryEnding` | 通关等大结局 |
| `Other` | 其它、测试或通用 |

**建议**：入口行的 **`Id`** 按 §3 命名，**`Timing Kind`** 与上表含义对齐，便于以后做配置校验工具。

## 7. 编辑器与 Addressables 检查清单

### 7.0 开场教学战对话（`TutorialDialogueIds`）

新档在 `tutorial_opening` 播完后会进入与「花」的教学战，以下入口 id 需在对话配置表中有对应行（缺行时控制器会打日志并跳过该段）：

- **`tutorial_battle_intro`**：进入战斗界面后首段对话，可配背景与立绘。
- **`tutorial_battle_force_king`**：强引导点国王前，可无背景。
- **`tutorial_battle_round3`**：第三回合出牌前，可无背景。
- **`tutorial_battle_defeat`**：教学战未胜（含平局）时单条战败提示，可无背景。

代码常量定义见 `Assets/Scripts/Models/TutorialDialogueIds.cs`。`BattleView` 预制体上可绑定 `tutorialHandDimmer`（第一回合遮罩）、`tutorialDefeatPlaceholder`（战败占位节点）；未绑定时对应效果跳过。

1. **对话表资源**：创建或复制 `DialogueConfig`，在 Addressables 中打上标签 **`config_dialogue`**，与 `AddressableLabels.ConfigDialogue` 一致。
2. **人物表资源**：创建 `HeroConfig` 与 `NpcConfig`，分别打上 **`config_hero`**、**`config_npc`** 标签；Hero 表需包含参赛者顺序与旁白 `0`，NPC 表的 `npcId` 需与塔层 `NpcIds` 一致。
3. **对话 UI**：`DialogView` 预制体绑定 `DialogView` 脚本各 `SerializeField`；Addressables 主地址 **`UI/DialogView`**，标签 **`ui_panel`**。
4. **表内 `Id`**：全项目合并后唯一；入口 id 与 §3 完全一致。
5. **立绘/背景**：人物表 `portraitId` / `avatarId` 与对话行 `Background Id` 能解析到已编址的 Sprite。
